Introduction
BDC Messenger is a modern, fullstack chat application developed as a major project for the Bachelor of Computer Application (BCA) program. The project demonstrates the integration of real-time communication, user authentication, group chat management, and responsive design using the latest web technologies.
Objectives
  • Enable secure, real-time messaging between users.
  • Support group chats with admin controls (add/remove members, update group info).
  • Provide user authentication and password management.
  • Ensure mobile-friendly and responsive user interface.
  • Showcase advanced features for college-level submission.
Technology Stack
  • Frontend: React (Vite), Zustand, DaisyUI, Tailwind CSS
  • Backend: Node.js, Express, MongoDB, Socket.io
  • Authentication: JWT, Cookies
  • Other: Cloudinary (profile images), Axios, PostCSS
Key Features
  • Email/password authentication and password reset flow
  • Real-time messaging and message deletion
  • Group chat with admin controls
  • Profile and settings management
  • Responsive/mobile-friendly UI for all devices
Future Implementation
  • Voice and video calling features for richer communication.
  • Push notifications for real-time alerts on mobile devices.
  • End-to-end encryption for enhanced privacy and security.
  • File sharing (documents, images, videos) within chats and groups.
  • Admin dashboard for user and group management.
  • Integration with college systems (attendance, announcements).
  • Progressive Web App (PWA) support for installable experience.
App Run Process
  1. Clone the project repository to your local machine.
  2. Install dependencies for both frontend and backend using npm install.
  3. Start the backend server: npm run dev in the backend folder.
  4. Start the frontend server: npm run dev in the frontend folder.
  5. Access the app in your browser at http://localhost:5174 (or the shown port).
  6. Register a new user, log in, and start chatting!
Deploy Process
  1. Build the frontend for production using npm run build in the frontend folder.
  2. Deploy the backend server to a cloud platform (e.g., Heroku, Render, or AWS).
  3. Deploy the frontend static files to a hosting service (e.g., Vercel, Netlify, or GitHub Pages).
  4. Set environment variables for backend (MongoDB URI, JWT secret, etc.).
  5. Configure the frontend to point to the deployed backend API.
  6. Test the deployed app for all features and mobile responsiveness.
Conclusion
This project fulfills the requirements of a fullstack chat application for academic submission, demonstrating practical skills in web development, user experience, and real-time systems. The application is ready for showcase and further enhancement.
